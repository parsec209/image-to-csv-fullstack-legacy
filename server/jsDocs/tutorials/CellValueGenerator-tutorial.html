This module contains the methods used for obtaining CSV cell section values from an uploaded file's text. Some of the methods are straightforward. For instance, the "pattern" method searches for text that matches a regular expression provided by the user. The "customValue" is just a literal value provided by the user, no text parsing necessary. The "today" method uses today's date (more specifically the date from the user's browser) as the value.

The positional "topPhrase" and "leftPhrase" methods are a bit more complicated. First, the term "phrase" has a specific meaning in the context of this application. It is a single line of text consisting of characters no more than one space apart from each other. A phrase ends when a character is proceeded by two or more spaces, a line break, or a page break. Furthermore, what we're using in these methods are "anchor phrases," meaning a phrase within a recurring document that has the following constants: 1) it is always in the same exact position of the document relative to the sought-after variable value, and 2) it is always the same phrase. The leftPhrase option uses an anchor phrase provided by the user that is known to always be located directly to the left of the variable value ("directly" meaning there are no nonspace characters in between the anchor phrase and the variable value). The anchor phrase will be contained in the text property of a line object (generated earlier in the TextLineGenerator module), and it will encompass either part of this line's text or all of it. The index of the line text where the phrase ends is important to note here; if there is additional text in the line after the phrase ends, the rest of that same line's text will be used as the cell section value. Otherwise, the cell section value will be obtained from the next line's text, which will be directly to the right of the line containing the anchor phrase.

The topPhrase uses an anchor phrase provided by the user that is known to always be positioned directly above the variable value ("directly" meaning it is first phrase above the variable value that shares an x-coordinate with it). Once again, this anchor phrase will be contained within the text property of a line object. Once that line is found, the app will then look for the first line directly below it that shares an x-coordinate with it. If said line exists, its text will be used as the cell section value.
